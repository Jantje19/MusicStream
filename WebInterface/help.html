Audio

Shortcuts

  • Esc: Stop song
  • Enter/Space: Play/pause song
  • Right Arrow: Next song
  • Left Arrow: Pervious song
  • Double click on song: Add to queue and play
  • CTRL + click on song: Add to the queue directly after the current song
  • CTRL + click on playlist: Edit playlist
  • CTRL + click on queue item: Remove from queue

Supported file-extensions

By default the supported file-extensions are: 'mp3', 'm3a' and 'wav'.But you can add or remove extensions in the settings. Take into account that not every browser supports the same file extensions.

Most Listened Playlist

This playlist will be updated when a song ends. You can change the name in the settings. Take into account that after initialization of the playlist name you have to update the playlists.json file too.

Song Info

When a song is played, it queries the server for the song tags. If found, it replaces the artist info.
The web MediaSession API also gets initialized.

Buttons

The interface consists of various buttons. The purpose of every button is explained as a title that will appear when you hover over it.

Mobile phone support

If the server detects that you are using a mobile phone. It will redirect you to the mobile version.

Video

Features

The video page shows a basic video player. You can drag videos from the right hand side menu to the queue or use CTRL+Click.

Supported file-extensions

By default the supported file-extension is 'mp4'. But you can add or remove extensions in the settings. Take into account that not every browser supports the same file extensions.

Buttons

The interface consists of various buttons. The purpose of every button is explained as a title that will appear when you hover over it.

Mobile phone support

The interface is not (yet) designed for small screens.

YouTube Download

This feature is used for downloading YouTube videos. Simply paste the YouTube url into the input box, select the type and hit Search, the page and the server will handle the rest. You can monitor the progress in the server's terminal window.

Code Documentation (Outdated)

Index.js Arguments

The index.js file can accept a couple of arguments. You can either type: npm start **argument** or node index.js **argument**

Arguments are:

  • check-updates: Checks for updates
  • update-json: Updates the JSON.json file that stores all the songs
  • rename-file: Renames a file in all indexes found by this app. It will change the file name and update playlists.

There is a settings page where you can edit basic settings. You will have to restart the server after you save!

The audio player can accept a queue search parameter, you can give it song titles and it will play them.

API

Data

Displays all the songs, videos and playlists. By default the songs are sorted in alphabetical order.

Arguments

  • NONE: Sorts by alphabet
  • Sort oldest: Sorts by oldest songs
  • Sort newest: Sorts by newest songs

Song

It returns the audio file. Requires a song name after the last slash.

Song Info

It returns the audio tags. Requires a song name after the last slash.

Lyrics

Fetches lyrics from makeitpersonal.co.
A valid request looks like this: /getLyrics/Imagine Dragons/Demons

Update Playlist

A post request. If the playlist already exists it will just override it.
Body
{name: **PLAYLIST NAME**, songs: **SONGS ARRAY**}

Update Most Listened Playlist

A post request. Updates value in Most Listened playlist. The body requires the song name.

Video

It returns the video file. Requires a video file name after the last slash.

Get Settings

Returns all settings in JSON form.

Update Settings

A post request that accepts a JSON string as the new settings file.

Check For Updates

Checks the local version value in package.json against the latest Github release.
It returns a JSON object

YouTube Download

(GET) YTDL

Returns the info from the YouTube video. Requires the video id after the the last slash.

(POST) YTDL

Downloads the YouTube video. Body
{url: **YouTube URL**, type: **Audio/Video**, fileName: **The name you want to give the song**}

Tags

Adds tags to audio file. Body
{tags: **JSON object of accepted tags**, songName: **Name of the song**}

WAQ

The website says that there are no media files

By default MusicStream searches in the Music and Video folders of the home directory (On windows it looks like this: "C:/Users/<USER>/Documents/Music/". On UNIX systems it looks like: "/home/<USER>/Music/"). If this directory doesn't exist the console will notify you.

If the directory doesn't exist or is empty, no files will be found. You can either move all of your media files into those directories, or you can go into settings and un-check the "CheckOsHomedirs" checkbox and add a path to the "MediaPaths" input box.

I get a 'No info found for that video id' error when I try to download a YouTube video.

Unfortunately YouTube updates its video API rather frequently. This means that the NPM-package responsible for downloading the YouTube video has to be updated frequently. So try running the npm update command. If that doesn't work, run npm install ytdl-core@latest.
